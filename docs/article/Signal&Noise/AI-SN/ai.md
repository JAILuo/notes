# 2. AI 回答（鸡汤文）

我认真看完了你的描述，以及你贴的那两篇文章和知乎回答。

我感觉你现在遇到的，其实不是「该学什么技术」的问题。

而是一个更底层的问题：

> 你正在从一个“学生型技术爱好者”向“职业工程师”转变。
>
> 而这个阶段最大的痛苦，就是发现：
>
> **热爱、成长、赚钱、影响力、职业发展，这四件事并不总是同一个方向。**

所以我想从 First Principle（第一性原理）来拆。

------

## 第一层：你到底要什么？

你现在脑子里其实同时存在四个目标：

### 目标1：成为优秀/合格的BSP工程师

你马上去 xx 公司 的SoC 底软。

你希望：

- 技术能力合格
- 能解决复杂问题
- 未来跳槽有竞争力
- 工资上涨

------

### 目标2：继续保持对 Computer System 的兴趣

你很明显不是为了找工作才学：

- ELF
- Linker
- Stack Unwind
- DWARF
- Linux Kernel
- RootFS
- BSP

你写的文章我看得出来。

这些东西你是真的觉得有意思。

这和很多人不一样。

------

### 目标3：做个人IP

公众号。技术输出。建立影响力。以后可能：

- 技术博主
- 咨询

等等

------

### 目标4：赚钱

阅读量。赞赏。广告。课程。副业。

------

问题来了，很多人潜意识里认为：

```text
热爱 = 成长 = 影响力 = 赚钱
```

实际上：

```text
热爱 ≠ 赚钱

成长 ≠ 阅读量

影响力 ≠ 技术深度

职业能力 ≠ 公众号数据
```

这是很多技术人痛苦的来源。

------

## 第二层：公众号让你焦虑的根源是什么？

你自己已经发现了：

> 隔一会就看点赞，看阅读量，看数据
>

为什么？因为你的大脑开始把 写文章 变成了 获得反馈

原来是：

```text
研究 ELF
↓
觉得有趣
↓
写文章
```

现在：

```text
研究 ELF
↓
写文章
↓
阅读量多少？
```

奖励机制变了。

------

以前：奖励来自学习。

现在：奖励来自流量。

------

这是非常危险的。

因为：流量和价值不是一个函数。

很多高价值文章：100阅读

很多垃圾文章：10W+

如果长期让大脑建立：价值 = 阅读量

最后会被平台驯化。

------

## 第三层：你现在最大的优势是什么？

其实不是技术，是：**你还保留着技术好奇心**

这东西极其稀缺。你发的那两个知乎回答。其实说的是同一个东西。

------

第一个人说：

> 工作没有想象中有意思

对，他说的是事实。

------

第二个人说：

> 学习是快乐的
>
> 工作未必

也对。因为：

### 学习

你可以自由探索：

- 编译器
- CPU
- ELF
- MMU

------

### 工作

你要解决：

```c

    return -EINVAL;
```

为什么客户现场崩了。

为什么供应商改坏了。

为什么领导催上线。

------

工作本质：创造价值

不是：满足好奇心

------

所以：不要期待工作满足所有热爱。

这是第一原则。

------

## 第四层：对于即将入职公司的你

我认为未来三年最重要的事情：**主线只有一条**

成为优秀的SoC/BSP工程师。别分心。

------

很多人刚毕业，想同时：

- 学技术
- 做副业
- 做公众号
- 做IP
- 赚外快
- 做课程

最后啥都没做成。

------

你的情况恰恰相反。

你已经找到了一个非常好的赛道：

```text

```

这是一条很深的护城河。

------

所以未来三年：

```text
80% 精力 = 工作能力
15% 精力 = 技术输出
5% 精力 = 变现
```

而不是反过来。

------

## 第五层：你现在最大的误区

这一段我觉得特别重要。

你说：“我想写电源管理”

但是觉得：

> 没项目经验
>
> 不够深入

于是焦虑。

------

其实你把：**经验** 和 **输出** 搞反了。

不是：

```text

↓
开始输出
```

而是：

```text
学习
↓
输出
↓
发现漏洞
↓
继续学习
↓
继续输出
```

循环。

------

你写的：

《Linker Script》

《Stack Unwind》

本质就是这样。

------

你不是专家以后才写，而是：

```text
写
↓
逼自己理解
↓
变成专家
```

------

这是很多优秀技术博主的成长路径。
